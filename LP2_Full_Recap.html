M01Azure Pipelines Foundation

Hierarchy

  • Pipeline → Stage → Job → Step
  • Jobs run on a single agent
  • Steps share the agent workspace
  • Cross-job = publish + download artifact

Triggers

  • trigger: — CI push
  • pr: — pull request
  • schedules: — cron
  • resources: pipelines: — pipeline
  • Manual — UI only

Service Connections

  • ARM → Workload Identity (best)
  • Docker Registry → images
  • GitHub → cross-repo
  • Project-scoped by default
🎯
Default stage condition = succeeded(). Classic pipelines cannot use RunOnce/Rolling/Canary strategies. Approvals live on the Environment, not the stage YAML.

M03Pipelines & Concurrency

Concurrency

  • batch: true — collapse rapid commits
  • lockBehavior: sequential — queue deployments
  • lockBehavior: cancelRunning — cancel older

Environment Checks

  • Manual approval
  • Azure Monitor alert
  • Azure Function / REST API
  • Query Work Items
  • Required template

Security

  • Protected resources
  • Secret variables (masked ≠ blocked)
  • Pipeline permissions
  • Retention: 30 days default
🎯
Approvals on the Environment, not the stage. batch: true = one run per batch of queued commits, not "run once".
M04Pipeline Strategy

Matrix Strategy

  • Same job × N variable combos
  • Parallel (needs parallel job budget)
  • maxParallel: caps concurrency

Branching

  • Trunk-based = exam preferred
  • GitFlow = versioned releases
  • batch: true for high-frequency

Artifacts & Cache

  • Pipeline artifacts = run-scoped temp
  • Azure Artifacts feeds = versioned
  • Cache key = hash of lock file
🎯
Pipeline artifact ≠ Azure Artifacts feed. Matrix + 1 parallel job = sequential runs, not parallel. Cache miss → fresh install → new cache saved.
M05YAML Integration & Templates

Template Types

  • Step / Job / Stage / Variable
  • Same repo or external repo
  • Required template = governance

Expression Syntax

  • ${{ }} compile-time
  • $[ ] runtime expression
  • $( ) macro (runtime)

Variable Groups

  • Shared across pipelines
  • Key Vault-linked = auto-rotation
  • Scope to pipelines that need them
🎯
${{ if }} = compile-time conditional — step is absent, not skipped. KV-linked variable group = secrets rotate in KV, zero pipeline changes needed.

M07CI with GitHub Actions

CI Job Order

  • checkout → setup toolchain → cache
  • → restore deps → build → test
  • → upload results (always)

Branch Protection

  • Required check = exact job name
  • Rename job = silent break
  • Set in Repo Settings → Branches

Caching

  • setup-node/python/java: built-in
  • actions/cache: manual key
  • Key = hashFiles(lock file)
🎯
if: success() || failure() on upload step — runs even when tests fail. Required status check name must match job name exactly.
M08Container Build Strategy

Multi-Stage Dockerfile

  • Stage 1: FROM sdk — build
  • Stage 2: FROM runtime — copy output only
  • No SDK in final image
  • Smaller, faster, fewer CVEs

ACR Tiers

  • Basic 10GB — dev/test
  • Standard 100GB — production
  • Premium 500GB — geo-replication + content trust

Best Practices

  • Tag with BuildId or git SHA
  • :latest as alias only
  • OIDC for ACR auth — no secrets
  • Docker service connection in ADO
🎯
ACR geo-replication + content trust = Premium only. Multi-stage = SDK never reaches production image. Always tag with specific immutable identifier for rollback capability.

🎯 Exam Watch — AZ-400

  • 🎯Q: Pipeline hierarchy top to bottom? → Pipeline → Stage → Job → Step.
  • 🎯Q: Which trigger fires when a PR is opened against main? → pr:
  • 🎯Q: Classic pipelines support RunOnce/Rolling/Canary? → No. YAML multi-stage pipelines only.
  • 🎯Q: Default condition on a stage? → succeeded() — not always().
  • 🎯Q: Best ARM service connection auth method? → Workload Identity Federation (passwordless, short-lived tokens).
  • 🎯Q: Which agent type is best for burst CI workloads that need zero idle cost? → VMSS (Azure VM Scale Sets) agent pool.
  • 🎯Q: Self-hosted agent vs Microsoft-hosted — key difference? → Self-hosted can reach private VNet resources. Microsoft-hosted cannot.
  • 🎯Q: batch: true on trigger — what does it do? → Waits for in-progress run to finish, then starts one run for all queued commits. Reduces parallel job consumption.
  • 🎯Q: Where do you add a human approval before a production deployment? → On the Environment resource in Azure DevOps. NOT in the YAML.
  • 🎯Q: Secret variable is masked in logs — can scripts still read it? → Yes. Masking is a display feature only.
  • 🎯Q: Pipeline artifact vs Azure Artifacts feed — which is permanent? → Azure Artifacts feed. Pipeline artifacts expire with run retention.
  • 🎯Q: Cache key changes when — what happens? → Cache miss. Full install runs, new cache is saved.
  • 🎯Q: compile-time conditional in YAML templates? → ${{ if }}. The step is absent from the run, not skipped.
  • 🎯Q: OIDC in GitHub Actions — what does it eliminate? → Stored cloud credentials in GitHub Secrets. Short-lived token per run instead.
  • 🎯Q: Required status check breaks after renaming CI job — why? → Required check matches by exact job name string.
  • 🎯Q: ACR tier for global teams needing geo-replication? → Premium.
